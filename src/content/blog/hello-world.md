## Why this site exists

Every developer eventually builds a personal site. This is mine. But I had two
constraints:

1. It must feel like a first-class macOS app. If Apple shipped a
   personal-website builder, this is how it would look.
2. It must be fast. Zero runtime JavaScript unless a feature genuinely
   needs it. The blog, projects, about — all static.

Astro hits both goals. It ships zero JS by default and gives me full control
over the CSS. No Tailwind, no UI library. Hand-written `backdrop-filter`
glass panels and CSS custom properties.

## The design language

Apple's Human Interface Guidelines for macOS emphasize:

- **Vibrancy** — translucent materials that adapt to what's behind them.
- **Depth** — subtle shadows and layering that create hierarchy without
  heavy borders.
- **Typography** — SF Pro, tight letter-spacing, generous line-height.
- **Motion** — spring physics, not linear easing.
- **Dark mode** — not an afterthought; every color adapts.

The site replicates all of these with CSS. No JavaScript framework needed.

## Color system

Everything is driven by CSS custom properties on `:root` and
`[data-theme="dark"]`. Light mode uses a `#f2f2f7` base (iOS/macOS system
background) with translucent white panels. Dark mode shifts to `#1c1c1e`
with translucent black panels.

```css
:root {
  --bg-glass: rgba(255, 255, 255, 0.82);
  --text-primary: rgba(0, 0, 0, 0.88);
}

[data-theme="dark"] {
  --bg-glass: rgba(36, 36, 38, 0.84);
  --text-primary: rgba(255, 255, 255, 0.92);
}
```

The glass effect itself is a single declaration:

```css
.card {
  background: var(--bg-glass);
  backdrop-filter: blur(24px);
  border: 0.5px solid var(--border);
}
```
