## 事情的起因

两三秒钟就能写完的东西，懒得多废话：说了一句话——「将本网站的板式风格基督教化，将各类措辞基督教化」。

## 发生了什么

Codex 自己就开始干活了。它先扫了一遍整个仓库的结构，搞清楚这是什么项目（Astro + GitHub Pages），然后自己列出了十二个待办事项，开始一个一个改。

CSS 里的强调色从橙色变成了哞哞金，背景从 iOS 灰变成了暖黄的羊皮纸色。导航栏里的「博客」变成了「默想」。首页标题从「你好，你就当我是个橘子」变成了「凡劳苦担重担的人，可以到我这里来」。关于页从一整篇 AI 生成讽刺文变成了使徒信经加圣经研读笔记。退出确认弹窗从「AI 魔法阵」变成了十字架加祝福语。

改了二十三个文件。构建零报错。推送成功。

从我说完那句话到它全部干完，中间没有任何需要我插手的地方。总共也就几分钟，大部分时间还是它自己 build 在等。

## 为什么好玩

不是因为它改得多么惊天动地——说实话改个配色加替换文案，任何前端工程师都能干。好玩的地方在于：**人类只需要表达意图，不需要描述方案**。

没告诉它该改哪个文件。没告诉它 CSS 变量叫什么名字。没告诉它 Astro 的 layout 系统怎么工作。没告诉它哪些文字在哪个组件里。它自己读代码、自己理解架构、自己写改动、自己验证构建。

这就是 LLM 在真实工程场景里的「现实扭曲力场」。以前：「我有一个想法 → 需要花半小时理解代码结构 → 花一小时改 → 花半小时检查」。现在：「我有一个想法 → 它好了」。
